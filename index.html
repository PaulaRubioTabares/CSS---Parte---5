Stacking context

El stacking context o contexto de apilamiento se refiere al orden en que se colocan o sobreponen los elementos o cajas que se crean en el html, teniendo en cuenta las propiedades que se coloquen.

El stacking content tiene la siguiente prioridad:

  1. Elemento con la propiedad de position y z-index de 1 ó más
  2. Elemento con la propiedad de position sin z-index
  3. Elemento sin la propiedad de position
  4. Elemento con z-index negativo

Propiedad de position y z-index de 1 ó más

Cuando se aplica la propiedad de z-index, el elemento por obligación debe llevar la propiedad de position.

Si hay dos elementos con position, pero uno de ellos tiene z-index positivo, este se colocará encima del otro.

Medidas relativas

Aunque ya antes he utilizado estas medidas sobre todo rem, es bueno saber cómo funcionan.

Rem

Esta medida relativa va en referencia al tamaño por defecto en html 16px.

Ejemplo

Texto 1 con tamaño del contenedor main.

Texto 2 con medida 2rem.

La medida del texto 2 "2rem" será en base a los 16px aunque al contenedor main de la página se le coloque un font-size diferente, en este caso de 1.5rem o 1.5(24px) = 24px.

Texto 2 2rem = 2(16px) = 32px.

Em

Medida relativa respecto a su contexto, los elementos tienen su tamaño respecto a sus contenedores.

Texto 1 con tamaño original.

Texto 2 con medida 2em.

En este caso, la medida dada si toma en cuenta los valores de tamaño definidos para la página, como se dijo antes, en el main de esta página se definió un font-size de 1.5rem o 24px, así que para calcular la medida de ese texto será con ese tamaño definido, no con la raíz del html.

2em = 2(1.5rem) = 2(24px) = 48px.

Width porcentual y auto

Otras medidas relativas son auto y porcentaje, más usadas en los tamaños de los contenedores.

Width %

Otra medida relativa es en % la cual se calcula automáticamente en relación con el tamaño de la ventana de visualización.

Caja con tamaño en 100% de su contenedor.

El espacio que queda de esta primera caja es del padding de su contenedor, pero al colocar un margin o padding al elemento que está dentro se puede desbordar.

Caja con tamaño en 100% de su contenedor y margin.

Este elemento ocupa a lo ancho un 100% de su contenedor, ese porcentaje no tiene en cuenta el espacio que se le coloque de margin al elemento que está dentro, por ello se sale de su contenedor y puede crear una barra de scroll.

Width Auto

Para solucionar el problema de desbordamiento en vez de colocar un width en % se coloca width auto, ya que así el tamaño lo calcula el navegador en base al contenido del elemento.

Caja con tamaño en auto de su contenedor y margin.

Con esto, se puede solucionar ese tipo de desbordamiento cuando se está usando width para ocupar todo el espacio disponible.

Height porcentual y auto

Estas medidas relativas al igual que se usan para width, se pueden usar para height.

Para colocar height %, si o si debe estar dentro de un contenedor con altura definida, si no está en ese contenedor, el valor que se coloque será reemplazado por height auto, y si no hay contenido en ese elemento entonces puede desaparecer.

Caja con height 50% dentro de un contenedor con height: 8em;

Cuando el elemento está dentro de un contenedor con alto definido, el height % de ese elemento será en base a la altura de su contenedor.

Caja con height 50% sin contenedor de altura definida.

Como este segundo elemento no está dentro de un contenedor con altura definida, no se aplica ningún % y se coloca el height en auto ocupando solo su contenido.

Y aunque no se vea, hay otro elemento con las mismas propiedades que los otros dos, pero al tener el height en %, y sin contenedor con height, este se coloca en auto y como no tiene contenido desaparece visualmente.

Viewport

Esta medida, toma como raíz el tamaño de la ventana del navegador o el área visible sin hacer scroll

Teniendo valores como:

  • Vw (Viewport width ó ancho del viewport)
  • Vh (Viewport height ó alto del viewport)
  • Vmin (Viewport mínimo)
  • Vmax (Viewport máximo)

Y para asignar un tamaño con esta medida se debe colocar, por ejemplo: height: 1vw; lo que equivaldría a 1% del alto de la ventana de visualización

Colores en Css

Para dar color a los elementos en css existen varios formatos:

Formato Ejemplo
Hexadecimal #CA6D94
RGB (red, green, blue) rgb(202, 109, 148)
RGBA (red, green, blue, opacidad) rgba(202, 109, 148, 0.687)
HSL (tono, saturación, luminosidad) hsl(335, 47%, 61%)

Los tres formatos sirven para dar la misma cantidad de colores.
