Stacking context

El stacking context o contexto de apilamiento se refiere al orden en que se colocan o sobreponen los elementos o cajas que se crean en el html, teniendo en cuenta las propiedades que se coloquen.

El stacking content tiene la siguiente prioridad:

  1. Elemento con la propiedad de position y z-index de 1 ó más
  2. Elemento con la propiedad de position sin z-index
  3. Elemento sin la propiedad de position
  4. Elemento con z-index negativo

Propiedad de position y z-index de 1 ó más

Cuando se aplica la propiedad de z-index, el elemento por obligación debe llevar la propiedad de position.

Si hay dos elementos con position, pero uno de ellos tiene z-index positivo, este se colocará encima del otro.

Más de un contexto de apilamiento

En los ejemplos anteriores solo había dos elementos, pero se pueden organizar más elementos y contenedores.

Elemento 1

Elemento h3 dentro de div

Esta práctica se compone de diversos elementos:

El primero es un div que es el contenedor general y es el fondo de todos los demás elementos.

El order en el html de los elementos que están dentro de ese contenedor es:

  1. Elemento 1
  2. Segundo div o segundo contenedor:
    1. Titulo h3
    2. Los párrafos
    3. La lista de elementos
  3. Elemento 2
  4. Elemento 3

En donde colocando propiedades como margin negativo, display, position, absolute, position relative y z-index se logra que cada elemento tenga un contexto de apilamiento.
